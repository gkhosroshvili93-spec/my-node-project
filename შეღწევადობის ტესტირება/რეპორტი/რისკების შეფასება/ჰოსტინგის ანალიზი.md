# (ჰოსტინგის ანალიზი)

## 1. ინსტრუმენტის მიმოხილვა
არ ვასახელებ

## 2. ექსპერიმენტის მიმდინარეობა

 გავუშვი ლოკალური სერვერი http ჰოსტინგზე და hack app (არ ვასახელებ) პარალელურად, რათა დამენახა რა ინფორმაცია გადიოდა S პროტოკოლის გარეშე

### 2.1. ფილტრაცია 

ქაოსის თავიდან ასაცილებლად, გამოვიყენე ფილტრი:
`http.request.method == "POST"`
ეს გვაჩვენებს მხოლოდ იმ პაკეტებს, სადაც მონაცემების გაგზავნა ხდება (მაგალითად, ლოგინ ფორმის შევსებისას).

## 3. ნაპოვნი მონაცემები 

რადგან ჩვენი სატესტო გარემო იყენებდა **HTTP** (და არა HTTPS) პროტოკოლს, მთელი ინფორმაცია იყო "Clear Text" (ღია ტექსტი) ფორმატში.
hack app-ის "Packet Details" ფანჯარაში, "HTML Form URL Encoded" სექციაში მკაფიოდ გამოჩნდა:

- **Form item: "username" = "admin"**
- **Form item: "password" = "Supertest123"**

## 4. საფრთხე 

ეს დემონსტრირება აჩვენებს, რატომ არის საშიში საჯარო Wi-Fi-ს გამოყენება (კაფეში, აეროპორტში). თუ საიტი არ იყენებს HTTPS-ს, ნებისმიერ ჰაკერს, რომელიც იმავე Wi-Fi-ზე ზის, შეუძლია ჩართოს ჰაკერული პროგრამები და მოიპაროს ყველა დაუშიფრავი მონაცემი.

## 5. რეკომენდაცია

- **HTTPS Only**: საიტმა უნდა გამოიყენოს SSL/TLS სერტიფიკატი.
- **HSTS Header**: სერვერმა უნდა "აიძულოს" ბრაუზერი, რომ ყოველთვის დაცული კავშირი გამოიყენოს.
